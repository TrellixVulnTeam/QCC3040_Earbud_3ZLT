Handles a USB HID Output/Feature Report from the host.

Generally called by an application's message handling task in response to receiving report data from the Host via the USB HID class.

Parameters
[in]Themessage's report id.
[in]Themessage's report size.
[in]Apointer to the message's report data.
Returns
The number of reports currently in the queue, will be non-zero if the report was queued successfully. Exact value can be used for tracking the current queue length, statistics / diagnostics etc.

◆ HidUpgradeRegisterInputReportCb()

void HidUpgradeRegisterInputReportCb ( hid_upgrade_input_report_cb_t  handler)

Register handler for report data going to the host (input reports).

The handler is called every time report data needs to be sent to the host. This is implemented as a callback function rather than a message, so that the data pointer can be accessed directly and is used immediately (avoids the need to buffer the data or add it to the message queue).
