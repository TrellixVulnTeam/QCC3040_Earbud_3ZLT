Detailed Description

Sent in response to requesting the advertising and scanning capabilities using the ConnectionDmBleGetAdvScanCapabilitiesReq() function.

This message is sent to the task specified in the above function.

This is a BT5.0+ message.

Field Documentation

◆ available_adv_sets

uint8 available_adv_sets

Number of advertising sets available to be used by application/profiles. 0 or 4 Note) If legacy controller then this will be 0.The number can vary at anytime due to resource limitations. This is unlikely.

◆ available_api

uint8 available_api

What AE subsystem APIs are allowed to be used. If bit set then API can be used. bit 0 - LE Legacy Advertising/Scanning API bit 1 - LE ExtAdv/ExtScan API bit 2 - LE PerAadv/PerScan API

Note) These are set to reflect what the controller can support and what LEGACY_ADV_SCAN_API_CONFIG has configured.

Note) MIB key legacy_hci_only can be forced to use legacy at build time via the build define AE_USES_LEGACY_HCI.

◆ max_periodic_sync_list_size

uint8 max_periodic_sync_list_size

The maximum number of periodic train sync records allowed to be stored by a periodic scanner. 0 to 3 Note) This information is used to allow a Periodic Scanner to sync on to 1 of the periodic trains. Refer to ConnectionDmBlePeriodicScanSyncTrainReq. The number can vary at any time due to resource limitations. This is unlikely.

◆ max_potential_size_of_rx_adv_data

uint16 max_potential_size_of_rx_adv_data

The potential max amount of advertising data or scan response data that can be processed by a scanner. Any advertising data larger than this will be thrown away. 31 to 1650 octets

Note) If legacy controller then this will be 31.

◆ max_potential_size_of_rx_periodic_adv_data

uint16 max_potential_size_of_rx_periodic_adv_data

The potential max amount of periodic advertising data that can be processed on the receive side of a periodic advertising train. Any periodic advertising data larger than this will be thrown away. 0 to 1650 octets

◆ max_potential_size_of_tx_adv_data

uint16 max_potential_size_of_tx_adv_data

The potential max amount of advertising data or scan response data that can be advertised. 31 to 251 octets

Note) This could be reduced depending on how an advertising set is configured (e.g. allow stack to use some of the space).

Note) If legacy controller then this will be 31.

◆ max_potential_size_of_tx_periodic_adv_data

uint16 max_potential_size_of_tx_periodic_adv_data

The potential max amount of periodic advertising data that can be advertised. This could be reduced depending on how an advertising set is configured (e.g. allow stack to use some of the space). 0 to 252 octets

◆ stack_reserved_adv_sets

uint8 stack_reserved_adv_sets

Number of advertising sets available to be used by upper stack. 0 or 1

Note) This is resource reserved to allow the stack to be able to advertise. It will only be used if instigated by another feature being enabled by the application (e.g. GAM).

◆ status

connection_lib_status status

0 = success, else error.

◆ supported_phys

uint16 supported_phys

Phys available to be used. If bit set then the phy is usable. bit 0 = LE 1M bit 1 = LE 2M bit 2 = LE Coded
